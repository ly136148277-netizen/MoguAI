# MOGU 2.3 Personal Remote Runbook

## Goal

Phone (Telegram first) → Remote Gateway → Permission → Task Queue → Agent/Skill → PC → notify phone.

No 2.4. QQ/WeChat stay OFF.

## 1. Create bot

Telegram → `@BotFather` → `/newbot` → copy `BOT_TOKEN`.

## 2. Store token (SecretStore only)

With desktop app running, from DevTools / internal IPC:

```js
await window.mogu?.invoke?.("remote:set-telegram-token", { token: "PUT_TOKEN_HERE" })
// or whatever preload bridge name exposes ipcRenderer.invoke("remote:set-telegram-token", ...)
```

Never put the token in `settings.json` or `.env`.

## 3. Bind owner + enable Telegram only

```js
await invoke("remote:configure-personal", {
  enabled: true,
  telegram: true,
  telegramUserId: "YOUR_NUMERIC_TELEGRAM_ID",
  requireApproval: true
})
```

Get your id via `@userinfobot` / `@getidsbot`.

Equivalent settings shape (token still SecretStore):

```json
{
  "remote": {
    "enabled": true,
    "telegram": { "enabled": true },
    "qq": { "enabled": false },
    "wechat": { "enabled": false },
    "requireApproval": true,
    "allowAutoExecute": false
  },
  "remoteOwner": { "telegramUserId": "YOUR_ID" }
}
```

## 4. Day-1 smoke

Phone:

```
/status
```

Expect GPU/task/model/queue lines. Strangers must be rejected.

Then Level-2:

```
/task 帮我检查项目状态
```

Dangerous:

```
删除文件 xxx
```

Must wait for YES/NO; without YES → no execute.

## 5. Field report

Fill `docs/V2.3_REMOTE_FIELD_REPORT.md` for 7 days. Then decide 2.4.
