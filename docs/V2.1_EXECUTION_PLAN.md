## Scope

2.1 adds four capability pillars without changing the frozen 2.0.x release line:

1. Repo intelligence: incremental file/symbol/import/reference/call indexes and controlled LSP.
2. Verified execution: test discovery and a permission-gated terminal.
3. Isolated exploration: at most two read-only worktrees and joined subtask results.
4. Recoverable runtime: append-only events, scoped leases, retry, checkpoints, and replay.

The GPT-5.6 Brain Adapter is an injected OpenAI-compatible contract. Model ID and endpoint are
configuration, not hard-coded facts. Missing GPT-5.6 configuration is `BLOCKED`; it must never
silently fall back to another model.

## Competitive capability method

2.1 is intentionally aggressive about capability learning. Publicly observable product behavior may
be measured, decomposed, specified, reproduced, combined and improved. License-compatible open-source
code may be used as a dependency, forked, ported or refactored with its notices and maintenance
obligations preserved. Proprietary implementation details use an independent behavior Spec and
clean-room implementation.

The prohibited boundary is unauthorized copying of proprietary source, private assets, internal data,
credentials or user secrets—not functional similarity. Every adopted capability must still enter
MOGU's permission, task, data and audit planes and prove its benefit under the same-model,
same-budget protocol.

## Trust boundary

Every external process is started by a MOGU-owned adapter and must pass:

```text
Brain → Runtime Adapter → PermissionProxy → Capability Interface
      → external process → Audit/Event Store → Verify
```

- Paths are canonicalized under declared roots. Symlink/junction escape is rejected.
- Terminal and LSP environments use explicit allowlists; secrets are not inherited.
- Read-only exploration cannot patch, commit, push, install, or update.
- L3 actions still require user confirmation. A lease cannot elevate risk level.
- Imported components cannot create another task store, secret store, updater, or telemetry path.

## Feature flags

All 2.1 flags default to `false`:

- `v21RepoIntelligence`
- `v21Lsp`
- `v21ControlledTerminal`
- `v21ParallelWorktrees`
- `v21RecoverableRuntime`
- `v21Gpt56Adapter`

Flags can be enabled separately for evaluation. No flag becomes default-on from development
results alone.

## Failure policy

- LSP crash: record event, stop server, allow one explicit restart; static index remains usable.
- Terminal timeout/cancel: terminate the process tree and emit a terminal event.
- Worktree failure: preserve event evidence, remove only manager-owned paths, never prune user worktrees.
- Retry: replay only a failed local step. A Gateway-accepted task is never automatically resubmitted.
- Event write failure: fail closed before the next side effect.
- Lease expiry/revocation/budget exhaustion: deny the action and record the denial.

## Done conditions

Each pillar requires unit tests, security edge cases, `acceptance:v2.1`, existing 2.0 gates, and a
machine-readable evaluation report. Development A/B must show the preregistered benefit twice
without safety regression before the sealed holdout is run. Holdout passing creates a candidate
only; final default-on promotion requires the owner discussion.

Full DAG scheduling, writable parallel agents, task migration, and global budget scheduling are
explicitly deferred to 2.3.
