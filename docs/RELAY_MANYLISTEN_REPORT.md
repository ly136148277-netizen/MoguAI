# 新家中转测评报告（manylisten）

**日期**：2026-07-19  
**对象**：`https://ai-api-router.manylisten.ccwu.cc/v1`  
**模型**：`gpt-5.6-sol`  
**MOGU 路径**：`cloud_openai_patch`（OpenAI 兼容 chat → SEARCH/REPLACE，未走 Codex 工具环）  
**对照 A**：本机 Ollama `qwen3:8b` + `local_ollama_patch`（`lite8-sr-20260719`）  
**对照 B**：Cursor 里的 Agent（本对话，多工具协作；**不是**同一套自动 SWE 流水线）  
**弃用**：旧家 `wecodex.lol`（欠费 / 宣传额度不可信）

> 本报告**不含 API Key**。过程指标 ≠ SWE 官方 Resolved %。  
> 「和 Cursor Agent 比」比的是**角色价值**（交互改码 / 无人值守批跑），不是同一条流水线的分数对打。

---

## 1. 结论（买了怎么办）

| 问题 | 判断 |
|------|------|
| 新家好不好用？ | **通道能用**；自动改码 8/8 有补丁；**官方 Resolved 0/8** |
| 和本机 8B 比？ | **补丁率更好、更慢、要花钱** |
| 和 Cursor Agent（我）比？ | **我更强、更会查案；新家适合塞进 MOGU 无人值守跑** |
| 钱花得值不值？ | **值在「产品里能自动跑」**；**不值在「想替代 Cursor 聊天体验」** |

**买了怎么用（实操）**

1. **留给 MOGU**：环境变量指向新家，`MOGU_CLOUD_PATCH=1`，批跑 / 工厂派工  
2. **难问题仍开 Cursor**：排查、定方案、修定位、写架构 —— 别指望中转一句顶我一轮工具链  
3. **本机 Ollama**：断网 / 省钱 / 小改动备用  
4. **旧家**：别充了  

**一句话**：新家是你产品的「云端工人」；我是「带工具的监工」。工人要留，监工别裁。

## 2. 三角对比：速度 / 回复 / 能力

### 2.1 回复速度（短问答，实测）

同一题：「用一句话解释什么是单元测试」各 3 次；另有 `PONG` 探针。

| 对象 | 短问答耗时（本次） | 极短 `PONG` | 体感 |
|------|-------------------|-------------|------|
| **新家 manylisten** `gpt-5.6-sol` | 6.1s / 6.3s / 21.8s（均 ~11s，冷启动偏慢） | 均 **6.6s**（P50 4.4s） | 能用，偏慢 |
| **本机 Ollama** `qwen3:8b` | 3.9s / 0.9s / 0.8s（热后 **~1s**） | 通常更快 | **对话最快** |
| **Cursor Agent（我）** | 短答通常 **1–3s** 出字（无工具）；一上工具就变成 **几十秒～数分钟** | 不适用纯 API | 交互快，重活慢 |

### 2.2 改代码端到端时间（同 8 道公开 SWE）

| 对象 | 非空补丁 | 单题典型耗时 | 说明 |
|------|----------|--------------|------|
| **新家 → MOGU 直出补丁** | **8/8**（首跑 6 + 修定位重跑 2） | ~30–90s（有的到 152s） | 无人值守，可批量 |
| **Ollama → MOGU 直出补丁** | **5/8** | ~20–30s | 免费但更易空改/错位 |
| **Cursor Agent（我）** | **未用同一自动 harness 交 8 题卷** | 本会话里查通/修链路是 **数十分钟级**（多轮工具） | 质量靠交互，不适合替你刷 300 题 |

### 2.3 能力（该找谁）

| 能力 | 新家+MOGU | Ollama+MOGU | Cursor Agent（我） |
|------|-----------|-------------|-------------------|
| 稳定 API / 鉴权 | ✅ | ✅ 本机 | ✅（Cursor 会话） |
| 无人值守批改仓库 | ✅ 为本设计 | ✅ 较弱 | ❌ 要人盯着聊 |
| 多文件排查、读日志、定方案 | 弱（当前直出补丁） | 更弱 | ✅ **明显更强** |
| 修中转/引擎挂起这类「元问题」 | 它自己就是被测对象 | — | ✅ |
| 官方 SWE Resolved | 未测 | 未测 | 未用同流水线交卷 |
| 成本 | 扣余额 | 电费 | Cursor 订阅 + 你的时间 |
| 替代 Cursor 聊天 | ❌ 更慢且无工具链 | ❌ | — |

### 2.4 直白对照（避免被一句话忽悠）

| 你的目标 | 该买/该用谁 |
|----------|-------------|
| 「MOGU 里自己跑题、派工改码」 | **新家值得**（相对旧家/8B） |
| 「聊天要快、本地免费」 | **Ollama** |

## 3. 连通与延迟（新家专项）

短回复 `PONG`（`max_tokens=8`），连续 5 次：

| 指标 | 结果 |
|------|------|
| 成功率 | 5/5 HTTP 200 |
| 最小 | 4.3 s |
| P50 | 4.4 s |
| 平均 | **6.6 s** |
| 最大 | 10.3 s |
| 总 token | 145（探针合计） |

解读：可用，但**不算快**；首包偶发 >10s。适合批跑，不适合当「毫秒级」体验卖点。

## 5. 对比总表（含 Cursor Agent）

| 维度 | 新家 manylisten | 本机 Ollama 8B | Cursor Agent（我） |
|------|-----------------|----------------|-------------------|

**相对 Ollama**：多 1 题非空补丁，更慢更贵。
